# FundMe Smart Contract

A beginner-to-intermediate Solidity project built using [Foundry](https://book.getfoundry.sh/?utm_source=chatgpt.com) for learning smart contract development, testing, and deployment.

This project allows users to fund the contract with ETH, while only the contract owner can withdraw the collected funds.

It is designed to practice real Web3 development workflows including:

* smart contract development
* deployment scripts
* professional testing
* debugging
* local blockchain deployment using Anvil

---

## Features

* Users can fund the contract with ETH
* Minimum funding requirement enforced
* Tracks each funder's contribution
* Owner-only withdrawal system
* Uses custom errors for gas-efficient access control
* Secure ETH transfer using `call()`
* Unit tests written using Foundry
* Local deployment using Anvil

## Smart Contract Overview

### `fund()`

Allows users to send ETH to the contract.

Requirements:

* minimum ETH must be sent

Stores:

* sender address
* funded amount

---

### `withdraw()`

Allows only the owner to withdraw all ETH from the contract.

Also:

* resets all funder balances
* clears the funders array

Uses:

* custom error `NotOwner()`
* low-level `call()` for safe ETH transfer

## Installation

Clone the repository:

```bash
git clone <your-repository-url>
cd fund-me
```

Install dependencies:

```bash
forge install
```

Build the project:

```bash
forge build
```

---

## Running Tests

Execute all tests:

```bash
forge test -vv
```

Tests include:

* owner initialization
* minimum funding validation
* funding data updates
* owner-only withdrawal protection
* successful withdrawal flow

---

## Local Deployment

Start local blockchain:

```bash
anvil
```

Deploy contract:

```bash
forge script script/DeployFundMe.s.sol \
--rpc-url http://127.0.0.1:8545 \
--private-key YOUR_PRIVATE_KEY \
--broadcast
```

Replace:

```text
YOUR_PRIVATE_KEY
```

with one of the private keys generated by Anvil.
